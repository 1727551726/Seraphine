# 大乱斗自动抢选英雄功能

## 功能概述

为 Seraphine 实现大乱斗（ARAM）模式自动抢选备选池英雄的功能，参考 LeagueAkari 项目的实现方式。

## 实现步骤

### 1. 配置项定义 (`app/common/config.py`)

新增 2 个配置项：
- `enableAramAutoSwap` — 大乱斗自动抢选开关
- `aramAutoSwapChampions` — 期望英雄列表（有序，优先级从高到低）

### 2. connector 修改 (`app/lol/connector.py`)

给 `benchSwap()` 方法添加 `@retry()` 装饰器，与其他 API 保持一致。

### 3. 抢选逻辑 (`app/lol/tools.py`)

新增：
- `AramBenchState` 类 — 备选席状态跟踪器，记录上次 bench 英雄集合和是否已执行抢选
- `autoBenchSwap()` 函数 — 自动抢选核心逻辑：
  1. 检查 `enableAramAutoSwap` 配置
  2. 检查 `benchEnabled`（大乱斗模式标志）
  3. 检查 `swapped` 标志（每轮只抢一次）
  4. 差异检测：比较当前 bench 与上次 bench，找出新上架英雄
  5. 按优先级匹配用户期望列表
  6. 调用 `connector.benchSwap()` 执行交换

### 4. 弹窗组件 (`app/components/message_box.py`)

新增 `AramBenchMsgBox` 类：
- 显示当前备选池英雄图标（5列网格）
- 自动抢选开关
- 状态显示标签

### 5. 设置卡片 (`app/view/gameflow_interface.py`)

新增 `AramAutoSwapCard` 类：
- 期望英雄选择（复用 `MultiChampionSelectMsgBox`）
- 启用开关
- 客户端未运行时的提示

在 `GameflowInterface` 中初始化并添加到 `bpGroup`。

### 6. 主窗口集成 (`app/view/main_window.py`)

- 导入 `autoBenchSwap`、`AramBenchState`、`AramBenchMsgBox`
- 初始化 `aramBenchState` 和 `aramBenchMsgBox`
- `__onChampionSelectBegin` 中添加弹窗逻辑
- `__onChampSelectChanged` 中添加自动抢选调用

## 遇到的问题及解决方案

### 问题 1: MultiChampionSelectWidget 中 connector.manager 为 None

**现象**: LOL 客户端未运行时，`connector.manager.getChampionNameById(id)` 抛出 `AttributeError`。

**原因**: `MultiChampionSelectWidget.__initWidget` 中调用了 `connector.manager.getChampionNameById(id)`，但客户端未运行时 `connector.manager` 为 `None`。

**解决**: 改用 `self.champions[id][0]` 获取英雄名称，`champions` 字典已包含名称信息。

### 问题 2: KeyError — 英雄 ID 不在 champions 字典中

**现象**: 配置中保存的英雄 ID 在当前英雄列表中不存在（版本不匹配等），导致 `KeyError`。

**解决**: 在访问 `self.champions[id]` 前添加 `if id not in self.champions: continue` 跳过不存在的 ID。

### 问题 3: 英雄搜索功能失效

**现象**: 输入英雄名称后没有搜索结果。

**原因**: `FlowLayout` 在隐藏/显示控件后没有自动触发布局重新计算。

**解决**: 在 `__onSearchLineTextChanged` 和 `__showAllChampions` 中添加 `self.championsShowLayout.invalidate()` 强制触发布局重算。

### 问题 4: 客户端未运行时选择英雄无提示

**现象**: LOL 客户端未运行时点击"选择"按钮，弹出空白的英雄选择界面。

**解决**: 在 `__onButtonClicked` / `__onSelectButtonClicked` 中添加 `self.champions` 为空检查，显示 InfoBar 警告"请先启动英雄联盟客户端"。

## 遗留事项

### 大乱斗模式开始时，未按照预期弹出小窗口显示当前可选英雄

**现象**: 进入大乱斗选人阶段时，`AramBenchMsgBox` 弹窗未自动弹出。

**可能原因**:
1. `__onChampionSelectBegin` 中的弹窗逻辑可能未被正确触发
2. `session.get('benchEnabled')` 在进入选人阶段时可能还未为 `True`
3. 弹窗创建后 `show()` 调用可能被后续的界面切换覆盖

**需要进一步排查**:
- 检查 `__onChampionSelectBegin` 的调用时机
- 确认 `benchEnabled` 字段在选人阶段开始时是否已设置
- 检查弹窗是否被创建但未显示

## 经验总结

1. **防御性编程**: 在访问字典前始终检查 key 是否存在，特别是跨版本的配置数据
2. **异步初始化**: 使用 `asyncio.create_task` 的任务如果抛出异常会被静默吞没，需要添加 try/except
3. **FlowLayout 更新**: 隐藏/显示控件后需要调用 `invalidate()` 触发布局重算
4. **信号连接**: 避免在初始化方法中重复连接信号，应在 `__initWidget` 中一次性连接
